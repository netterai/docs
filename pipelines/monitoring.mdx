The **Monitoring** page (`/monitoring`) is where you watch pipeline health across every project you have access to. Two views share the same data.

## Overview

The overview answers "what's currently running, what failed, what's scheduled?" at a glance:

- **Step stats** — totals for active, draft, paused, archived.
- **Dashboard stats** — totals for ready, draft, generating, error.
- **Runs (last 7 days)** — counts by status: success, failed, running, cancelled.
- **Recent runs** — the most recent executions across all projects with status, trigger source, and elapsed time.
- **Recent errors** — failed runs with the error summary.
- **Active triggers** — every scheduled or polling trigger with its next-fire timestamp.

Everything respects project permissions—you only see projects you're a viewer or higher on.

## Insights

The insights view turns the raw data into a small set of actions you can take now.

| Block | What it shows |
|-------|---------------|
| **Status banner** | A health score (0–100) and a banner: healthy (≥80), warning (≥50), or critical. |
| **Priority actions** | The few highest-impact things to fix—reconnect an expired integration, resolve a timed-out step, clear a stuck schedule. |
| **Smart insights** | Patterns the platform notices automatically (slow step, repeating failure, stale dataset). |
| **Activity timeline** | The last 48 hours of run successes, failures, deploys, and trigger errors. |

## When to use which

- Pick **Overview** when you want raw status: "is anything failing right now?"
- Pick **Insights** when you want guidance: "what should I fix first?"

## Project-level history

The Monitoring page is company-wide. For a single pipeline's history—runs, cascade events, per-step rows processed—open the project and use its **Run history** tab. See [Runs](/pipelines/runs).
